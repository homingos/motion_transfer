# R2 Mode (Dev Environment)

## Overview
This is the **avatar ID mode** - you send an avatar ID and the system finds the image in R2, generates the video, and uploads it back to R2.

**URL**: `https://ai-team-flam-dev--motion-transfer-dev.modal.run`

## How It Works (Plain Terms)

1. You provide an **avatar ID** (a unique identifier for an avatar)
2. System looks up the avatar in MongoDB
3. Finds the image stored in R2 (cloudflare storage)
4. Generates the motion video
5. Uploads the result back to R2
6. Returns you a request ID to check status

## API Endpoints

### 1. Check Avatar Info
**Get avatar details without generating**

```bash
curl https://ai-team-flam-dev--motion-transfer-dev.modal.run/avatar/507f1f77bcf86cd799439011/info
```

Response:
```json
{
  "avatar_id": "507f1f77bcf86cd799439011",
  "status": "ready",
  "image_key": "avatars/123/source.jpg",
  "image_exists_in_r2": true,
  "idle_animation_key": "templates/507f1f77bcf86cd799439011/idle",
  "idle_animation_exists_in_r2": true
}
```

### 2. Generate Motion Video
**Submit a generation job with avatar ID**

```bash
curl -X POST https://ai-team-flam-dev--motion-transfer-dev.modal.run/idle-motion \
  -F "avatar_id=507f1f77bcf86cd799439011"
```

Response:
```json
{
  "request_id": "abc123def456",
  "status": "pending",
  "avatar_id": "507f1f77bcf86cd799439011",
  "submitted_at": "2024-06-16T14:30:00Z"
}
```

### 3. Check Job Status
**Check if your generation is done**

```bash
curl https://ai-team-flam-dev--motion-transfer-dev.modal.run/jobs/abc123def456
```

Response (while running):
```json
{
  "status": "running",
  "avatar_id": "507f1f77bcf86cd799439011",
  "started_at": "2024-06-16T14:30:05Z"
}
```

Response (when done):
```json
{
  "status": "done",
  "finished_at": "2024-06-16T14:35:45Z",
  "animation_key": "templates/507f1f77bcf86cd799439011/idle",
  "link": "https://r2-url.com/templates/507f1f77bcf86cd799439011/idle"
}
```

### 4. Download Result
**Get your generated MP4 video**

```bash
curl https://ai-team-flam-dev--motion-transfer-dev.modal.run/jobs/abc123def456/result \
  --output result.mp4
```

### 5. Update Avatar Status
**Manually set avatar status (useful for error recovery)**

```bash
curl -X POST https://ai-team-flam-dev--motion-transfer-dev.modal.run/avatar/507f1f77bcf86cd799439011/status \
  -F "status=ready"
```

Valid statuses: `processing`, `ready`, `failed`

## Step-by-Step Example

```bash
# 1. Check if avatar exists and image is available
curl https://ai-team-flam-dev--motion-transfer-dev.modal.run/avatar/507f1f77bcf86cd799439011/info

# 2. Submit generation job
REQUEST_ID=$(curl -X POST https://ai-team-flam-dev--motion-transfer-dev.modal.run/idle-motion \
  -F "avatar_id=507f1f77bcf86cd799439011" | jq -r '.request_id')

echo "Submitted job: $REQUEST_ID"

# 3. Wait and check status (keep polling)
for i in {1..60}; do
  STATUS=$(curl https://ai-team-flam-dev--motion-transfer-dev.modal.run/jobs/$REQUEST_ID | jq -r '.status')
  echo "Status: $STATUS"
  if [ "$STATUS" = "done" ]; then
    break
  fi
  sleep 5
done

# 4. Download result
curl https://ai-team-flam-dev--motion-transfer-dev.modal.run/jobs/$REQUEST_ID/result \
  --output my_video.mp4

echo "Done! Video saved as my_video.mp4"
```

## Parameters

### Generate Motion Video (`/idle-motion`)
- **avatar_id** (required) - The unique ID of your avatar
- **lora_strength** (optional) - Control strength (0.0-1.0, default 0.8)
- **video_strength** (optional) - Motion control (0.0-1.0, default 0.85)

## Notes

- Image uploads are **NOT allowed** in this mode
- The avatar must exist in MongoDB with `source_assets.image_key`
- Generation takes 2-5 minutes
- Videos are stored in R2 permanently
